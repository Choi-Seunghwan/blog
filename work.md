현재 개인 개발 블로그 이전을 검토 중이다.

## 배경

* 지금은 **Tistory**로 개발 블로그를 운영 중이다.
* 하지만 Markdown 작성/렌더링 경험이 만족스럽지 않고, 전체적인 UI/타이포/문단 인상이 다소 아쉽다.
* 나는 **Notion을 개인 메인 워크스페이스**로 계속 사용할 예정이며, 아이디어 정리, 문서 정리, 일정 관리 등은 계속 Notion에서 한다.
* 다만, Notion에서 보던 문단/레이아웃 감성이 Tistory로 옮기면 많이 죽는다고 느낀다.
* 개발 블로그는 단순 기록 공간이 아니라, **개발자 역량과 취향을 보여주는 공개 포트폴리오 성격**도 있다고 생각한다.
* 그래서 “없어 보이지 않는”, 좀 더 **개발자스러운 인상의 블로그 UI**를 원한다.
* 완전히 세련된 SaaS 랜딩 느낌보다는, 오히려 **문서형 / 미니멀 / 터미널풍 / 코드블록이 예쁜** 쪽이 더 취향이다.
* “구식 UI인데 오히려 진짜 개발자 같아 보이는 느낌”도 선호한다.

## 방향 결정

* 결론적으로 **개인 도메인 기반의 독립 블로그**를 만드는 쪽으로 가고 싶다.
* Tistory는 점점 한계가 있다고 느끼고 있다.
* 공개 블로그는 내 이름의 개발자 사이트처럼 보이길 원한다.

## 후보 스택

* **도메인**: 개인 도메인 사용
* **블로그 프레임워크**: **Astro**
* **UI/테마 후보**: **Astro Terminal**

  * 터미널풍 / 모노스페이스 / 레트로 개발자 감성
  * 개인 개발자 블로그 느낌이 강해서 마음에 든다

## Astro 블로그에 대해 이해한 내용

* Astro로 블로그를 구축할 때는 보통:

  1. 테마(예: Astro Terminal)로 시작
  2. 내 정보/스타일/레이아웃 수정
  3. `astro build` 로 정적 파일 생성
  4. 정적 사이트처럼 배포
* 즉, 일반적인 정적 프론트엔드 배포처럼 다룰 수 있다.

## 내가 생각 중인 배포 방식

* **S3 + CloudFront** 구조로 배포하는 방식을 생각 중이다.
* 즉,

  * 로컬에서 수정
  * 빌드
  * `dist` 산출물 생성
  * S3 업로드
  * CloudFront로 서빙
* 내가 이해한 것이 맞는지 확인하고 싶다.

## 글 발행 방식에 대한 이해

* CMS처럼 웹에서 직접 글 쓰는 방식이 아니라,
* 보통은 **Markdown/MDX 파일을 Git으로 관리**하는 구조로 이해하고 있다.
* 즉:

  * 새 글 작성 = 새 markdown 파일 추가
  * 수정 = 파일 수정
  * 발행 = git commit / push 후 빌드 및 배포
* 이 방식이 일반적인 Astro 블로그 운영 방식인지 확인하고 싶다.

## 원하는 도움

다음 내용을 기준으로 설명/구현 가이드를 받고 싶다.

1. **Astro Terminal을 실제로 어떻게 시작하는지**

   * 템플릿 설치/클론 방식
   * 초기 설정
   * 내 정보 및 스타일 수정 포인트

2. **Astro 블로그의 일반적인 운영 구조**

   * 글 디렉토리 구조
   * Markdown/MDX 작성 방식
   * frontmatter 관리
   * 태그, 날짜, 목록 페이지 구성 방식

3. **배포 방식 검토**

   * `Astro build -> S3 -> CloudFront` 흐름이 적절한지
   * 일반 정적 FE 배포처럼 운영하면 되는지
   * 실무적으로 권장되는 배포 구조가 무엇인지

4. **발행 프로세스 설계**

   * 글 작성부터 발행까지의 표준 흐름
   * git commit 기반 발행 방식이 일반적인지
   * GitHub Actions 등으로 자동 배포하는 것이 좋은지

5. **내 상황에 맞는 추천 아키텍처**

   * Notion은 내부 작업실로 유지
   * 공개 블로그는 개인 도메인의 Astro 블로그
   * 필요 시 Notion의 초안을 Markdown으로 정리해 발행하는 워크플로우

## 목표

* “글을 쓰는 공간”과 “공개 발행되는 개발자 블로그”를 분리하고 싶다.
* 내부 작업은 계속 Notion에서 하되,
* 외부에는 **개발자답고 보기 좋은 개인 기술 블로그/포트폴리오 사이트**를 운영하고 싶다.
* 너무 무거운 CMS보다는, **정적 사이트 + Git 기반 발행**의 단순하고 개발자스러운 운영 방식을 선호한다.
